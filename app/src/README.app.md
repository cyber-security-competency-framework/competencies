# Cyber Security Competencies (CSeC) Framework

This project contains the definition of a skills and competencies framework to help us classify and describe technical and non-technical skills we use in our different roles as security engineers and analysts, security operations specialists, risk and compliance specialists etc. (collectively known as the  CySPs - Cyber Security Professionals), along with lists of tools/methods/behaviours to demonstrate skills and training materials and resources that can help us gain a particular skill.

It also includes a project to implement a webapp to enable practical use of the framework - to browse the skills and competencies, self-assess and create individual skill profiles as “competency wheels”, compare profiles across a team, define template skills for different digital roles and other use cases.

## Defining and classifying skills

Skills and competencies are described in this ReadMe file and we have a [GitHub issue to track suggestions](https://github.com/cyber-security-competencies-framework/issues/1).

If you think skills are missing, needs clarification, or could be reorganised within the framework, please [create a PR] (https://github.com/cyber-security-competencies-framework) or add your feedback to the [GitHub issue to track suggestions](https://github.com/cyber-security-competencies-framework/issues/1).

## The Framework

### Proficiency
The compentencies are broken down into five proficiency levels:

1. Novice
2. Advanced Beginner
3. Practitioner
4. Proficient
5. Expert

### Competencies:

#### Cyber Resilience
**Detect, respond to, and learn from issues and incidents and anticipate attacks.**
- Security Monitoring
- Incident & Issue Response
- Threat and Internal Context

#### System Hardening
**System level building and testing to prevent, find, and fix exploitable weaknesses and to make attacks less likely, less impactful and more recoverable.**
- System & Application Security
- Security Architecture
- Identity and Access
- Data Security

#### Secure the Organisation
**Security at scale, implementing control loops, process improvements and amplifying individual contribution with workflows and automation, ensuring that the organisation is not only secure but also adaptable, efficient and resilient.**
- Governance, Risk and Compliance
- Security in Contracts
- Security Process Engineering
- Organisational Resilience
